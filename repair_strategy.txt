Method: Changed the static variable declaration in the fuzzer driver function fuzz_parse_http_request() in lwan-request.c from 'static struct coro_switcher switcher;' to 'static coro_context switcher;' to match the type change introduced by the root cause commit.

Reasoning: The ECRCL root cause analysis identified commit 6c0c7fb17a54e1780f009b447c2020cbb918b11d ("Remove indirection when swapping coroutine context") as the source of the compilation failure. This commit removed the 'struct coro_switcher' definition from lwan-coro.h and changed all internal references from 'struct coro_switcher *' to 'coro_context *' across lwan-coro.c, lwan-template.c, and lwan-thread.c. However, the fuzzer driver function fuzz_parse_http_request() in lwan-request.c (guarded by #ifdef FUZZING_BUILD_MODE_UNSAFE_FOR_PRODUCTION) was missed during this refactoring and still declared 'static struct coro_switcher switcher;'. Since 'struct coro_switcher' no longer exists as a defined type (only a forward declaration remains), the compiler emitted "variable has incomplete type 'struct coro_switcher'". Changing the declaration to 'static coro_context switcher;' aligns it with the new type used everywhere else. The subsequent call 'coro_new(&switcher, ...)' remains correct because '&switcher' on a coro_context variable yields a 'coro_context*', matching the updated coro_new() signature.
